Welcome to Workout API



Get started


Workout API is an API that helps you manage your workout tracking app. It provides you with user profile management, and authentication. It also provides you with a variety of exercises that the user can subscribe to, and tracks users' records day by day.




Endpoints




New user:

POST: http://workout-track-api.herokuapp.com/api/v1/users/sign-up

It does accept three parameters::name, password, birth_date

A simple request may appear like:

fetch( 'http://workout-track-api.herokuapp.com/api/v1/users/sign-up', {

headers: { 'Content-Type': 'application/json' },

method: 'POST',

body: JSON.stringify({name: 'Kola', password: 'xxx', birth_date: 'Some date here.'}), },

)




User login:

POST: http://workout-track-api.herokuapp.com/api/v1/users/sign-in

It does accept two parameters::name, password

A simple request may appear like:

fetch( 'http://workout-track-api.herokuapp.com/api/v1/users/sign-in', {

headers: { 'Content-Type': 'application/json' },

method: 'POST',

body: JSON.stringify({name: 'Kola', password: 'xxx'}), },

)




User update profile:

POST: http://workout-track-api.herokuapp.com/api/v1/user/profile

It does accept two parameters::weight, height

A simple request may appear like:

fetch( 'http://workout-track-api.herokuapp.com/api/v1/user/profile', {

headers: { 'Content-Type': 'application/json' },

method: 'POST',

body: JSON.stringify({weight: '75', height: '178'}), },

)




User untracked exercises:

GET: http://workout-track-api.herokuapp.com/api/v1/user/${:id}/untracked-movements

It does accept one parameter::id

A simple request may appear like:

fetch( 'http://workout-track-api.herokuapp.com/api/v1/user/1/untracked-movements')




User tracked exercises:

GET: http://workout-track-api.herokuapp.com/api/v1/user/${:id}/tracked-movements

It does accept one parameter::id

A simple request may appear like:

fetch( 'http://workout-track-api.herokuapp.com/api/v1/user/1/tracked-movements')




User records:

GET: http://workout-track-api.herokuapp.com/api/v1/user/${:id}/records

It does accept one parameter::id

A simple request may appear like:

fetch('http://workout-track-api.herokuapp.com/api/v1/user/1/records')




User top score in a specific exercise:

GET: http://workout-track-api.herokuapp.com/api/v1/user/:id/:movement_id/topscore

It does accept two parameter::id, movement_id

A simple request may appear like:

fetch('http://workout-track-api.herokuapp.com/api/v1/user/1/2/topscore')




Add exercise to user tracked exercises:

POST: http://workout-track-api.herokuapp.com/api/v1/movements/new

It does accept two parameters::user_id, movement_id

A simple request may appear like:

fetch( 'http://workout-track-api.herokuapp.com/api/v1/movements/new', {

headers: { 'Content-Type': 'application/json' },

method: 'POST',

body: JSON.stringify({user_id: 1, movement_id: 3}), },

)




Add a new record of a movement or exercise:

POST: http://workout-track-api.herokuapp.com/api/v1/user/records/new

It does accept three parameters::user_id, movement_id, movement_count

A simple request may appear like:

fetch( 'http://workout-track-api.herokuapp.com/api/v1/user/records/new', {

headers: { 'Content-Type': 'application/json' },

method: 'POST',

body: JSON.stringify({user_id: 1, movement_id: 3, movement_count: 60}), },

)
